## 1. 技术栈

| 组件 | 说明 |
|------|------|
| **Jest** | 测试运行器与断言（`package.json` 中 `jest` 字段） |
| **ts-jest** | 将 TypeScript 测试编译后在 Node 中执行 |
| **`@nestjs/testing`** | 构造 `TestingModule`、覆盖 Provider；当前单测以 Service 直接 `new Service(mock)` 为主，可按需扩展 |
| **Supertest** | 已在 `devDependencies`；用于「启动 HTTP」的集成/e2e 场景 |

**Jest 配置要点**：

- `rootDir`：`src`，默认只拾取 `src/` 下的 `*.spec.ts`。
- `testRegex`：`.*\.spec\.ts$`。
- 覆盖率输出目录：相对 `rootDir` 为 `../coverage`，即 **`backend/coverage/`**（HTML：`coverage/lcov-report/index.html`）。

---

## 2. npm 脚本

| 命令 | 作用 |
|------|------|
| `npm test` | 运行全部单元测试 |
| `npm run test:watch` | 监听模式 |
| `npm run test:cov` | 全量单测 + 覆盖率（`collectCoverageFrom` 覆盖整个 `src` 匹配项） |
| `npm run test:cov:tasks` | 仅 tasks 模块用例 + 仅统计 `src/modules/tasks/**/*.ts` 覆盖率 |
| `npm run test:tasks` | 仅跑路径含 `modules/tasks` 的用例（无覆盖率） |
| `npm run test:e2e` | 端到端，使用 `test/jest-e2e.json`（**非**默认单测配置） |
| `npm run test:debug` | Jest 断点调试 |

> **注意**：Jest 30+ 筛选路径请使用 **`--testPathPatterns`**，旧参数 `--testPathPattern` 已移除。

---

## 3. 目录与文件约定

| 类型 | 位置 | 说明 |
|------|------|------|
| 领域单测 | 与源文件同目录，如 `src/modules/tasks/tasks.service.spec.ts` | 与被测类一一对应，便于维护 |
| 可复用 Test helper | `src/**/__tests__/` 或 `test/helpers/` | 团队择一；勿与 `*.spec.ts` 命名冲突 |
| e2e | `test/e2e/*.e2e-spec.ts` | 独立 Jest 配置 |

---

## 4. Mock 策略

- **Prisma / `PrismaService`**：对需用到的 delegate（如 `client.findMany`、`user.findUnique`）使用 **`jest.fn()`** 做浅层 mock，**不连接真实数据库**；断言关注业务分支、`where`/`data` 形状及是否调用/未调用某方法。
- **其它 Nest 依赖**（如 `OrganizationContextService`、`JwtService`、`ConfigService`）：用 **`jest.fn().mockResolvedValue(...)`** 或 `as unknown as XService` 注入最小假实现。
- **与 e2e 分工**：单测验证 Service/小单元逻辑；全链路、真实路由与（可选）真实库放在 **`test:e2e`**。
