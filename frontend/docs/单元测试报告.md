**生成日期**：2026-04-30

## 1. 技术栈

| 组件 | 说明 |
|------|------|
| **Vitest** | 与 Vite 同构的测试运行器；`npm test` 为 watch，`npm run test:run` 为单次 CI 式执行 |
| **@vitest/coverage-v8** | V8 覆盖率，与 Vitest 主版本对齐 |
| **jsdom** | Node 内模拟 DOM，供组件测试使用 |
| **@testing-library/react** | 以可访问性/角色查询组件 |
| **@testing-library/jest-dom** | 扩展断言（如 `toBeInTheDocument()`），在 `vitest.setup.ts` 注册 |
| **@testing-library/user-event**（可选） | 模拟点击、键盘等交互 |

**配置要点**：

- `environment`：`jsdom`  
- 路径别名 **`@` → `src`**，与开发构建一致  
- 覆盖率：`test.coverage` 中 `provider: 'v8'`，`reportsDirectory` 为项目下 **`frontend/coverage/`**（HTML：`coverage/index.html`）

---

## 2. npm 脚本

| 命令 | 作用 |
|------|------|
| `npm test` | Vitest **watch** 模式 |
| `npm run test:run` | **单次**跑完全部单测（适合 CI / 提交前） |
| `npm run test:tasks` | 仅 `src/features/tasks` 下用例，不收集覆盖率 |
| `npm run test:coverage` | 全量单测 + 覆盖率 |
| `npm run test:coverage:tasks` | 仅 tasks 用例 + 覆盖率，且 `coverage.include` 限定为 `src/features/tasks/**` |

---

## 3. 目录与文件约定

| 类型 | 位置 | 说明 |
|------|------|------|
| API / 纯逻辑 | 与源文件同目录，如 `api.test.ts` 与 `api.ts` 并列 | 命名 **`*.test.ts`** |
| 页面 / 组件 | 同目录 **`*.test.tsx`** | 由 Vitest `include` 收集 |
| 全局 setup | 仓库根 **`vitest.setup.ts`** | 与 `vite.config.ts` 中 `setupFiles` 一致 |

---

## 4. Mock 策略（简述）

- **`features/*/api.ts`**：对 `@/lib/api/http` 使用 **`vi.mock`**，将 **`apiRequest`** 设为 **`vi.fn()`**，用 **`mockResolvedValue`** 断言路径、`method`、`body`、**`headers`**（如 `X-Organization-Id`），**不发起真实 fetch**。
- **带 TanStack Query 的页面**：使用 **`QueryClientProvider`**，可调低 **`queries.retry`**；可对具体 `api` 模块做 **`vi.mock(..., importOriginal)`** 局部替换。
- **纯函数**（如 `groupAssignableRowsToTrackProjects`）：无需 mock HTTP，直接断言输入输出。
