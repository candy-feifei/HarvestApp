# HarvestApp 前端目录结构说明

便于分工：按 **领域（feature）** 认领模块；公共能力放在 **`src/lib`** 与 **`src/components`**；**不要**在业务页直接散落 `fetch` 与魔法字符串路径。

---

## 1. `frontend/` 根目录（工程级）

```
frontend/
  docs/
    前端搭建说明.md      ← 搭建过程与决策
    目录结构说明.md      ← 本文档
  public/                 ← 静态资源（favicon 等）
  src/                    ← 应用源码（见第 2 节）
  components.json         ← shadcn 配置（别名、样式入口等）
  eslint.config.js
  index.html
  package.json
  package-lock.json
  tsconfig.json             ← 含 paths，供 shadcn / IDE 解析 @
  tsconfig.app.json
  tsconfig.node.json
  vite.config.ts            ← React 插件、Tailwind 插件、@ 别名
```

## 3. 路径别名

| 别名 | 指向 |
|------|------|
| `@/*` | `src/*` |

示例：`import { Button } from '@/components/ui/button'`。

---

## 4. 协作约定（简版）

1. **新页面**：优先落在对应 `features/<领域>/pages/`；路由在 `app/router.tsx` 用 `lazy` 指向该页导出组件。
2. **新接口**：在对应 `features/<领域>/api.ts` 中封装，内部使用 `apiRequest`；列表/详情优先用 **TanStack Query**（`useQuery` / `useMutation`）。
3. **新通用组件**：仅当跨 2 个及以上领域复用时，抽到 `src/components/`（非 `ui`）；单领域专用组件放在该 `features/<领域>/components/`（目录可按需新建）。
4. **导航**：新增菜单项时修改 `lib/nav-config.ts`，并同步 `router.tsx` 中的 lazy 与 `lazyModuleRoot` 映射，避免「有菜单无路由」。
5. **鉴权**：业务壳层由 `RequireAuth` 包裹；公开页仅 `/login`。令牌仅存 `sessionStorage`，勿写入 localStorage；新接口默认走 `apiRequest` 即可附带 Bearer。

|------------------|----------------------|---------------------|
| auth（登录） | — | `/login`（不在侧栏） |
| dashboard | dashboard | `/` |
| organization | organization | `/organization` |
| projects | projects | `/projects` |
| tasks | tasks | `/tasks` |
| time | time | `/time` |
| reports | reports | `/reports` |
| expenses | expenses | `/expenses` |
| access | access | `/access` |
| settings | settings | `/settings` |

Nest 侧 `*Module` 命名可与上表 id 对齐，便于评审与代码走读。
