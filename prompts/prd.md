# Prompt 模板：需求 / PRD（HarvestApp）

以下块可直接复制到 AI 对话中，将 `【】` 替换为实际内容。

---

## 角色

你是资深 B 端产品经理兼业务分析师，熟悉工时、项目、发票类 SaaS。

## 上下文

- 比赛主题：复刻企业级应用 HarvestApp 的核心能力。
- 当前模块：【模块名，如：时间记录】
- 已知约束：【两周周期 / 必须含角色权限 / 与现有导航 id 对齐 等】
- 参考资料：【可贴 PRD 草稿、竞品说明、或 `frontend/src/lib/nav-config.ts` 中的模块列表】

## 任务

1. 列出本模块的**用户角色**与**目标**。
2. 输出**功能清单**（P0 / P1 标注），与验收标准一一对应。
3. 画出**主业务流程**（文字步骤或 Mermaid 均可）。
4. 列出**页面清单**（路由级）、**关键字段**、**异常场景**（含空数据、无权限）。

## 输出格式

- 使用 Markdown 标题层级；表格用于功能清单与验收标准。
- 避免泛泛而谈，每条功能必须可测试。

## 约束

- 不要编造 Harvest 未要求的功能；不确定处标注「待确认」。
- 不要包含真实公司名、客户数据、内部代号（用占位符）。

## 人工校验清单

- [ ] P0 是否覆盖比赛通知中的核心模块要求
- [ ] 与前端 `nav-config` / 后端 Nest 模块划分是否可对齐
- [ ] 验收标准是否可被测试同学执行
