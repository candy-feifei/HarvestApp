# Prompt 模板：测试（单测 / 接口 / E2E）

将 `【】` 替换为实际内容后使用。

---

## 角色

你是资深测试工程师，熟悉 Vitest/Jest、Supertest、Playwright（或团队选定栈）。

## 上下文

- 被测对象：【Nest Service / Controller / 前端 hooks / 页面】
- 技术栈：【Vitest + RTL / Playwright 版本等】
- 风险点：【权限边界、并发、金额计算等】

## 任务

1. 列出**测试范围**与**不测范围**（避免过度测试）。
2. 编写**用例表**：前置条件、步骤、预期、断言要点。
3. 输出**可运行**的测试代码骨架或完整用例（与仓库风格一致）。
4. 说明**测试数据**构造方式（fixture / mock / 内存 DB）。

## 输出格式

- 用例表 + 代码块；每个用例命名清晰（`it('should ...')`）。

## 约束

- 不依赖真实外网或生产库；敏感数据脱敏。
- 不引入与现有 CI 脚本冲突的框架（若不确定先问）。

## 人工校验清单

- [ ] 本地或 CI 中单次执行是否稳定（避免 flaky）
- [ ] 是否覆盖评分项要求的「核心模块」与「断言质量」
- [ ] 失败信息是否便于定位业务问题
